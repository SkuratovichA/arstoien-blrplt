- **Why it wasn't needed:**
  - PostgreSQL Docker image automatically creates the database and user from `POSTGRES_USER`, `POSTGRES_PASSWORD`, and `POSTGRES_DB` environment variables
  - The init.sql was redundant - it tried to create the same database/user that Docker already creates
  - Extensions like `uuid-ossp` and `pgcrypto` can be added through Prisma migrations if needed
- Removed the init.sql mount from docker-compose.yml

**To run the server:**
```bash
# 1. Start Docker services
docker-compose up -d

# 2. Generate Prisma client
cd packages/server
npx prisma generate

# 3. Run migrations
npx prisma migrate dev

# 4. Seed database (optional)
npx prisma db seed

# 5. Start server
yarn dev
```
